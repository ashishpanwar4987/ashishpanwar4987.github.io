# Publishing Your Website on GitHub Pages

Your site is one file: `index.html`. Publishing takes about 10 minutes.

## Steps

1. **Create a GitHub account** (if you don't have one): github.com → Sign up. Choose a clean username, e.g. `ashishpanwar`.

2. **Create the repository**: click **+** (top right) → *New repository*. Name it exactly `<username>.github.io` (e.g. `ashishpanwar.github.io`). Keep it Public. Click *Create repository*.

3. **Upload the file**: on the repository page, click *uploading an existing file*, drag in `index.html`, click *Commit changes*.

4. **Done.** After 1–2 minutes your site is live at `https://<username>.github.io`.

## Updating the site later

Open `index.html` in the repository → click the pencil icon → edit → *Commit changes*. Or edit the copy in this folder and re-upload it. (You can also ask Claude to make the edits and give you the updated file.)

## Custom domain (optional, later)

Buy `ashishpanwar.com` (~US$10–12/yr from Namecheap or Cloudflare), then in the repository: *Settings → Pages → Custom domain*. GitHub's docs walk through the two DNS records needed.
